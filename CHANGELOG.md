## 既知のバグ・制限事項

| 状態 | 内容 |
|------|------|
| 未対応 | 再請求CSVに含まれるターミナルケア加算が当月（再請求月）に記録される可能性がある。本来は算定開始月として登録されるべき |
| 制限 | CSVはRMシステムのSEI_RM形式のみ対応。他社システムのCSVフォーマットには非対応 |
| 制限 | パスワードはクライアント側ハッシュ比較のため、ソースを見れば突破可能。機密情報の管理には不向き |
| 制限 | GASの1セルあたり50,000文字上限があるため、1事業所・1ヶ月の利用者数が極端に多い場合（100名以上）は保存に失敗する可能性がある |

---

## 諦めたこと・技術的負債

| 項目 | 理由 |
|------|------|
| 認証の強化（ログイン・権限管理） | GASとGoogle Sheetsで権限管理するには複雑度が高くなりすぎるため見送り |
| 複数ユーザー同時編集の競合検出 | 実運用上ほぼ1人が操作するため優先度低 |
| テストコードの整備 | 単一HTMLファイルの構成上、ユニットテストの導入コストが高い |
| グラフライブラリの導入（Chart.js等） | 外部依存なしでSVGで実装することで、オフライン・CDN障害時も動作するよう維持 |
